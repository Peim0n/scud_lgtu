# Архитектура проекта

## Общая схема

```
┌─────────────────────────────────────────────┐
│                  app.py                      │
│   Главный цикл + настройка логирования       │
└──────────────────┬──────────────────────────┘
                   │
┌──────────────────▼──────────────────────────┐
│         AccessController                     │
│   Бизнес-логика, авторизация, события        │
└──────┬──────────────────┬───────────────────┘
       │                  │
┌──────▼──────┐  ┌────────▼─────────┐
│  ScudEngine │  │   ApiServer      │
│  GPIO/потоки│  │   HTTP API       │
└──────┬──────┘  └──────────────────┘
       │
   ┌───┴───┬──────────┬──────────┐
   │       │          │          │
   ▼       ▼          ▼          ▼
PinCtrl  Wiegand   Serial    Signals
(магистраль I/O)
```

## Модули

- `scud_lgtu.engine` — управление железом и потоками.
- `scud_lgtu.access_controller` — логика доступа, обработка событий.
- `scud_lgtu.api_server` — HTTP API для тестирования.
- `scud_lgtu.pin_controller_thread` — мультиплексор и сдвиговый регистр в одном потоке.
- `scud_lgtu.passage_detector` — обработка датчиков прохода.
- `scud_lgtu.wiegand_reader` — чтение Wiegand-карт.
- `scud_lgtu.serial_reader` — чтение серийных портов.
- `scud_lgtu.local_access_cache` — локальный кэш разрешённых идентификаторов.
- `scud_lgtu.identifier_hash` — хеширование идентификаторов.
- `scud_lgtu.qr_encoder` / `qr_decoder` — кодирование и декодирование QR.

## Конфигурация

Все тайминги, ключи, логирование и API находятся в `config.yml`.
